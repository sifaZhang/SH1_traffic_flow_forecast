# Dataset

## Data Name

TMS traffic quarter-hourly: Jan 2013 to Sept 2020(https://opendata-nzta.opendata.arcgis.com/datasets/b719083bbb09489087649f1fc03ba53a/about)

## Data Source

New Zealand Transport Agency

# Project Structure

- `data/` stores the preprocessed data files for model training and evaluation, not included in the Git repository due to large size.
- `models/` stores the trained model weights and preprocessing objects, not included in the Git repository.
- `notebooks/` analysis and training notebooks
- `results/` stores the output charts and evaluation results

# Project running prerequisites

The `data/` folder contains preprocessed data files generated by the notebooks.
These files are excluded from version control due to their large size.

## How to Generate the `data/` Folder

1. Download the raw traffic data from New Zealand Transport Agency (NZTA) Traffic Monitoring System.

2. Place the CSV files in the following folders:
   - `../../traffic_data/old_format/` (2013-2020, TMSTrafficQuarterHour format)
   - `../../traffic_data/new_format/` (2020-2021, tms_YYYY_MM format)

3. Run the notebooks in order:
   - `notebooks/01_EDA.ipynb` → generates `cleaned_2078.csv`, `cleaned_2079.csv`
   - `notebooks/02_preprocessing.ipynb` → generates all `.npy` and `.pkl` files
